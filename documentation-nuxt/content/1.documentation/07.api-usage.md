## How It Works

::callout{icon="i-lucide-info" color="warning"}
There are no real API requests to a server. The application reads data directly from these static JSON files. You can modify or extend them as needed.
::

To use real data instead of mock JSON files, create reusable API helpers and fetch data with Vue composables. This keeps components clean and allows easy switching to real REST endpoints.

## Integration Steps

::steps

### Create a Product Data Composable

Create a composable to fetch products with axios:

```ts [src/shared/composables/useProducts.ts]
import { ref } from "vue";
import axios from "axios";

export const useProducts = () => {
  const products = ref([]);
  const loading = ref(true);
  const error = ref<string | null>(null);

  const fetchProducts = async () => {
    loading.value = true;
    try {
      const res = await axios.get("/api/products.json");
      products.value = res.data;
    } catch (err: any) {
      error.value = err.message;
    } finally {
      loading.value = false;
    }
  };

  fetchProducts();

  return { products, loading, error };
};
```

### Use the Composable in a Component

Now you can use your composable in any Vue component to display products:

```vue [src/widgets/ProductList.vue]
<script setup lang="ts">
import { useProducts } from "@/shared/composables/useProducts";

const { products, loading, error } = useProducts();
</script>

<template>
  <div v-if="loading">Loading...</div>
  <div v-else-if="error">Error: {{ error }}</div>
  <ul v-else>
    <li v-for="p in products" :key="p.id">{{ p.name }}</li>
  </ul>
</template>
```

## Summary

::callout{icon="i-lucide-lightbulb" color="info"}
The example above demonstrates how to fetch product data using a Vue composable.  
For best practices, create separate composables for each type of API request (such as categories, orders, reviews, etc.) to keep your codebase clean, modular, and scalable.
::
