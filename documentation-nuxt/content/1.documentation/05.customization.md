The project supports full customization including styling via SCSS variables and asset management.

## Global Styles

Global styles and SCSS variables are located in the `src/app/styles` folder. This is where you control the overall look and feel of the application.

![SCSS Folder](/images/03.png)

### Edit CSS Variables

You can customize the appearance of the entire application by editing SCSS variables in any file within the `src/app/styles` folder. These variables define colors, spacing, fonts, and other design properties, allowing you to easily adapt the theme to your brand or preferences.

For example, in `src/app/styles/_variables.scss` you might see:

```css [src/css/variables.css]
$white-color: #fff;
$main-yellow: #f5c102;
$titles-color: #222222;
$tomato-color: #fe724e;
$main-color: #0f1620;
$main-color2: #e83939;
```

## Local Assets

The project uses a minimal set of local assets. Most images are loaded via external URLs for easy customization.

::callout{icon="i-lucide-file-image" color="info"}
The project contains only one local asset file located at `src/assets/01.png`. All other images are loaded via external URLs defined within the Next.js `Image` component.
::

## How to Change Images

::steps

### Replace a Local Image

If you want to replace the local image (`src/assets/01.png`), simply put your new image file in the same location and use the same filename. This will update the image everywhere it is used in the project.

### Replace an Image URL

If the image is loaded via an external URL, find the `Image` component in the source code and swap the existing URL with your own:

```tsx
// Before
<Image src="https://old-image-url.com/photo.jpg" alt="description" />

// After
<Image src="https://your-new-url.com/photo.jpg" alt="description" />
```

::
