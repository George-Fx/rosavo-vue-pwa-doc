The project supports full customization including styling via SCSS variables and asset management.

## Global Styles

Global styles and SCSS variables are located in the `src/app/styles` folder. This is where you control the overall look and feel of the application.

![SCSS Folder](/images/03.png)

### Edit SCSS Variables

You can customize the appearance of the entire application by editing SCSS variables in any file within the `src/app/styles` folder. These variables define colors, spacing, fonts, and other design properties, allowing you to easily adapt the theme to your brand or preferences.

For example, in `src/app/styles/_variables.scss` you might see:

```scss [src/app/styles/_variables.scss]
$white-color: #fff;
$main-yellow: #f5c102;
$tomato-color: #fe724e;
$main-color: #0f1620;
$gainsboro-color: #dadfe2;
$text-color: #6c757d;
$accent-color: #ff4343;
$alabaster-color: #efede6;
$platinum-color: #e2e2e2;
$cerulean-frost-color: #60a2d5;

$border-radius: 10px;
$header-height: 50px;
$screen-width: 600px;
$tab-bar-height: 76px;
```

## Scoped Styles

In addition to global styles, each screen component has its own local styles defined inside a `<style scoped>` block at the bottom of the `.vue` file. Scoped styles apply only to the current component and do not affect other parts of the application.

```vue [src/pages/Home.vue]
<style scoped>
.container {
  padding: 16px;
  background-color: var(--background-color);
}
</style>
```

::callout{icon="i-lucide-info" color="info"}
Scoped styles are the recommended place for component-specific overrides. Use global SCSS variables for shared design tokens and `<style scoped>` for layout or element-level adjustments unique to a screen.
::

## Local Assets

The project uses a minimal set of local assets. Most images are loaded via external URLs for easy customization.

::callout{icon="i-lucide-file-image" color="info"}
The project contains only one local asset file located at `public/thumbnail.png`. All other images are loaded via external URLs referenced directly in `<img>` tags throughout the Vue source files.
::

## How to Change Images

::steps

### Replace a Local Image

If you want to replace the local image (`public/thumbnail.png`), simply put your new image file in the same location and use the same filename. This will update the image everywhere it is used in the project.

### Replace an Image URL

If the image is loaded via an external URL, find the `<img>` tag in the relevant `.vue` file and swap the existing URL with your own:

```vue
<!-- Before -->
<img src="https://old-image-url.com/photo.jpg" alt="description" />

<!-- After -->
<img src="https://your-new-url.com/photo.jpg" alt="description" />
```

::
