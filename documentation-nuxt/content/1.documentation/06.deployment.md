Choose one of the following methods to deploy your project:
manual deployment from the `dist` folder, or automated deployment using platforms like Vercel or Netlify.

## Method 1: Deploy from dist Folder (Manual/Custom Server)

::steps

### Build the project

Before deploying, create an optimized production build:

```bash [Terminal]
npm run build
```

After the build, all output files are located in the `dist` folder.

### Deploy the Project

Upload the contents of the `dist` folder to your hosting. Several options are available:

:::card{icon="i-lucide-server" title="Static File Server"}
Serve the `dist` folder with any static file server (nginx, Apache, etc.).
:::

:::card{icon="i-simple-icons-docker" title="Docker"}
Containerize your app with a Docker image for consistent deployments across environments.
:::

:::card{icon="i-lucide-file-code" title="CDN"}
Upload static files to any CDN (AWS S3 + CloudFront, Cloudflare Pages, etc.).
:::

::

## Method 2: Deploy with Vercel or Similar Platforms

::steps

### Push to Git Repository

Push your project to a Git provider:

- [GitHub](https://github.com/)
- [GitLab](https://gitlab.com/)
- [Bitbucket](https://bitbucket.org/)

### Connect to Hosting Platform

Connect your repository to a deployment platform:

- [Vercel (Recommended)](https://vercel.com/)
- [Netlify](https://netlify.com/)
- [Render](https://render.com/)
- [Firebase Hosting](https://firebase.google.com/docs/hosting)

### Configure Build Settings

Set the following in the platform's build settings:

| Setting          | Value           |
| ---------------- | --------------- |
| Build command    | `npm run build` |
| Output directory | `dist`          |

### Deploy and Go Live

Follow the platform instructions to trigger the deployment process.  
Your project will be built and published automatically.

::
