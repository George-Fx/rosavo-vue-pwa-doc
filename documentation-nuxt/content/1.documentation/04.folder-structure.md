The project is structured following the **[Feature-Sliced Design (FSD)](https://feature-sliced.design/)** methodology — an architectural approach that organizes code by business features and layers, improving scalability and maintainability. Learn more in the [official FSD documentation](https://feature-sliced.design/docs/get-started/overview). Here's an overview of each folder and its purpose.

![Folder Structure](/images/02.png)

## Directory Overview

:::card{icon="i-lucide-app-window" title="src/app/"}
Contains the main application setup: root component, global styles, router and store initialization.
:::

:::card{icon="i-lucide-database" title="src/entities/"}
Contains business entities (e.g. product, cart item) — data models, types, and entity-level components.
:::

:::card{icon="i-lucide-layout-template" title="src/pages/"}
Contains page-level components that correspond to application routes.
:::

:::card{icon="i-lucide-blocks" title="src/shared/"}
Contains shared utilities, UI primitives, constants, and helpers reused across the project.
:::

:::card{icon="i-lucide-puzzle" title="src/widgets/"}
Contains large self-contained UI blocks composed from entities and shared components (e.g. header, product card).
:::

:::card{icon="i-lucide-file-type" title="src/main.ts"}
Application entry point — mounts the Vue app and registers global plugins.
:::
