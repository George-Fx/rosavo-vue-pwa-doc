The project is structured following the **[Feature-Sliced Design (FSD)](https://feature-sliced.design/)** methodology — an architectural approach that organizes code by business features and layers, improving scalability and maintainability. Learn more in the [official FSD documentation](https://feature-sliced.design/docs/get-started/overview). Here's an overview of each folder and its purpose.

![CSS Folder](/images/02.png)

## Directory Overview

:::card{icon="i-lucide-app-window" title="src/app/"}
The top-most FSD layer. Contains application-wide setup: global providers, router configuration, global styles, and the root component that bootstraps the entire app.
:::

:::card{icon="i-lucide-box" title="src/entities/"}
Business entities of the domain (e.g. `User`, `Product`, `Order`). Each entity encapsulates its own model, UI, and API calls. Entities can only import from `shared`.
:::

:::card{icon="i-lucide-sparkles" title="src/features/"}
User-facing interactions and use-cases (e.g. `auth`, `add-to-cart`). Features compose entities and shared code to implement specific product scenarios.
:::

:::card{icon="i-lucide-layout-dashboard" title="src/pages/"}
Full pages of the application. Each page assembles widgets, features, and entities into a complete route-level view.
:::

:::card{icon="i-lucide-share-2" title="src/shared/"}
Reusable infrastructure shared across all layers: UI kit, utility functions, constants, TypeScript types, API clients, and other cross-cutting concerns. Has no dependencies on other FSD layers.
:::

:::card{icon="i-lucide-layout-panel-top" title="src/widgets/"}
Large, self-contained UI blocks composed of features and entities (e.g. `Header`, `Sidebar`, `ProductCard`). Widgets are used by pages.
:::

:::card{icon="i-lucide-file-type-2" title="src/main.ts"}
Application entry point. Initializes the Vue app instance, registers plugins, and mounts the root component to the DOM.
:::
